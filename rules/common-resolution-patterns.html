Common Resolutions

Most challenges in Rivendawn follow recognizable patterns. Whether characters oppose one another, assist an ally, overcome environmental hazards, or undertake extended effort, these common situations are resolved using the same unified engine that governs the rest of the game. These patterns do not create new subsystems; they simply clarify how the core resolution rules apply in everyday play, ensuring that cooperation, conflict, and sustained effort remain fast, consistent, and easy for the Watcher to adjudicate.

I. Purpose

While the Dice Resolution System defines how actions are resolved, certain situations arise frequently enough to warrant standardized guidance.

The following patterns describe common ways characters interact with opposition, cooperation, environmental challenges, and sustained effort.

All patterns use the unified Dice Resolution System described in the Dice Resolution System section.

Resolution always follows the same mechanical framework:

Capability = Primary Attribute + Ability/Knowledge + Context Rating

Target Number = Baseline − Capability

Roll 1d20.

Success occurs when the roll equals or exceeds the Target Number.

II. Opposed Actions

Opposed actions occur when two characters directly attempt to overcome one another.

Both participants roll using the appropriate Capability.

Resolution follows these outcomes:

  • If only one participant succeeds, that participant prevails.
  • If both succeed, the higher roll determines the winner.
  • If both fail, the situation remains unresolved.

Opposed actions represent simultaneous effort rather than sequential attempts.

Typical examples include stealth versus awareness, deception versus insight, or grappling versus escape.

III. Assistance

One character may assist another in performing a task.

The assisting character rolls using the appropriate Capability.

  • If the assistance roll succeeds, the primary actor receives −2 to their Target Number.
  • If the assistance roll fails, no benefit is granted.

Only one assistance attempt may apply to a single action.

Assistance represents coordination, guidance, or supporting effort rather than shared execution.

IV. Group Actions

Group actions occur when multiple characters attempt the same task simultaneously.

Each participant rolls individually.

  • If half or more of the participants succeed, the group succeeds.
  • If fewer than half succeed, the group fails.

Group actions represent coordinated effort rather than individual performance.

Typical examples include forcing open a gate, searching a large area, or climbing a difficult surface together.

V. Environmental Difficulty

Environmental conditions may increase the difficulty of an action.

The Watcher may increase the Target Number when the environment introduces resistance or complication.

Typical modifiers include:

Minor Difficulty
+2 to Target Number

Severe Conditions
+4 to Target Number

Extreme Circumstances
+6 to Target Number

Environmental difficulty represents external resistance rather than character limitation.

Examples include unstable footing, violent weather, darkness, hazardous terrain, or obstructed visibility.

VI. Repeated Attempts

Repeated attempts occur when a character tries the same action multiple times under unchanged circumstances.

Each additional attempt increases the Target Number by +1.

This penalty accumulates with each subsequent attempt.

The modifier resets once circumstances meaningfully change.

Repeated attempts represent growing difficulty, fatigue, or diminishing opportunity.

VII. Extended Tasks

Some actions require sustained effort rather than a single successful roll.

Extended tasks measure progress through accumulated successes.

Each successful roll contributes 1 Progress.

The Watcher determines the required progress threshold based on task complexity.

Typical progress thresholds include:

Simple Task — 2 successes

Moderate Task — 4 successes

Major Task — 6 successes

Extended tasks represent gradual accomplishment rather than immediate resolution.

Examples include dismantling complex mechanisms, stabilizing critically injured allies, or breaking through reinforced barriers.

VIII. Design Principle

These resolution patterns provide structure for common gameplay situations while remaining fully compatible with the Dice Resolution System.

They exist to simplify adjudication without introducing new mechanical subsystems.

All actions within Heroes Resonance ultimately resolve through the same unified engine.

The Song remains lawbound.

These patterns simply describe the situations where its laws most often apply.
